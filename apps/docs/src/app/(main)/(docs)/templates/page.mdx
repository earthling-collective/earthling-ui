# Templates

Earthling UI provides a variety of project templates to help you get started quickly. These templates are pre-configured with Earthling UI and follow best practices for their respective project types.

## Available Templates

### Application Templates

Application templates are designed for building user-facing applications.

| Template Name            | Command                                         | Features                                                                                                                                                                    | Aliases          |
| ------------------------ | ----------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------- |
| Next.js (`next`)         | `bunx earthling-ui create next my-app`          | Server-side rendering, File-based routing, API routes, Pre-configured with Earthling UI components, Tailwind CSS integration, TypeScript support                            | `ssr`, `nextjs`  |
| Vite SPA (`vite`)        | `bunx earthling-ui create vite my-spa`          | Fast development server, Optimized production build, Pre-configured with Earthling UI components, Tailwind CSS integration, TypeScript support                              | `spa`            |
| Tauri Desktop (`tauri`)  | `bunx earthling-ui create tauri my-desktop-app` | Cross-platform desktop application, Small binary size, Native system integration, Pre-configured with Earthling UI components, Tailwind CSS integration, TypeScript support | `app`, `desktop` |
| Electron Kiosk (`kiosk`) | `bunx earthling-ui create kiosk my-kiosk-app`   | Fullscreen kiosk mode, Electron runtime, Pre-configured with Earthling UI components, Tailwind CSS integration, TypeScript support                                          | `electron`       |

### Package Templates

Package templates are designed for building libraries and utilities.

| Template Name           | Command                                     | Features                                                                               | Aliases               |
| ----------------------- | ------------------------------------------- | -------------------------------------------------------------------------------------- | --------------------- |
| Database Package (`db`) | `bunx earthling-ui create db my-db-package` | Drizzle ORM integration, Database migrations, TypeScript support, Ready for publishing | `database`, `drizzle` |
| Library Package (`lib`) | `bunx earthling-ui create lib my-library`   | TypeScript support, Testing setup, Documentation scaffolding, Ready for publishing     | `library`             |
| CLI Package (`cli`)     | `bunx earthling-ui create cli my-cli`       | Command parsing, Interactive prompts, TypeScript support, Ready for publishing         |                       |

### Repository Templates

| Template Name         | Command                                          | Features                                                                                      | Aliases |
| --------------------- | ------------------------------------------------ | --------------------------------------------------------------------------------------------- | ------- |
| Monorepo (`monorepo`) | `bunx earthling-ui create monorepo my-workspace` | Workspace configuration, Shared dependencies, Organized project structure, TypeScript support | `repo`  |

## Best Practices

1. **Choose the right template** for your project type
2. **Keep dependencies up to date** to benefit from the latest features and security fixes
3. **Follow the established patterns** in the template for consistency
4. **Contribute improvements** back to the Earthling UI project if you find ways to enhance the templates
